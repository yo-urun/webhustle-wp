Основная проблема с WooCommerce Setup Wizard (онбординг-визардом) — это распространённая неисправность, особенно с кастомными темами. Визард использует современный WC Admin (React-based интерфейс на странице /wp-admin/admin.php?page=wc-admin&path=/setup-wizard или похожей), который очень чувствителен к глобальным стилям, JavaScript и REST API. Бесконечная загрузка или нерабочие кнопки («Настроить магазин» / «Пропустить настройку») обычно вызваны ошибками JS, неудачными REST-запросами (404/500/405), конфликтами CSS/JS или серверными настройками.
Ключевые причины на основе поиска в интернете

Конфликты темы/плагинов: Кастомная тема (особенно с Tailwind, Alpine.js, глобальными enqueue) часто ломает WC Admin. Стили/скрипты, загружающиеся в админке, переопределяют React-компоненты, блокируют события кликов или вызывают бесконечный спиннер.
REST API проблемы: Ошибки на эндпоинтах вроде /wp-json/wc-admin/onboarding/profile или /wp-json/. Причины — неправильные пермалинки, отсутствие rewrite-правил (.htaccess/web.config), серверные ограничения (IIS/Windows, Nginx), security-плагины (Wordfence и т.п.).
JS-ошибки: Посмотрите консоль браузера (F12 → Console/Network) на странице визарда — ищите ошибки (TypeError, failed to fetch, sprintf и т.п.). Alpine.js или Tailwind-классы часто конфликтуют.
Другие: Проблемы с сервером (index.html вместо index.php), кэш, незавершённая установка WC, geolocation/SSL. Иногда помогает ручной skip визарда.

Плагин в вашем случае — вероятно, сам WooCommerce (визард запускается при активации). Если это WooCommerce Memberships, то он обычно не имеет своего визарда, но может взаимодействовать с core WC.

Ваш DOCS.MD хорошо описывает интеграцию:

wh_woocommerce_setup — объявление поддержки WC + сброс дефолтных стилей (важно, чтобы не дублировались).
Кастомные обёртки (woocommerce_before_main_content / after), side-cart (Alpine + AJAX-фрагменты), мини-карта.
Поддержка Memberships в inc/memberships.php: бейджи на продуктах/в лупе, кастомные рестрикшен-сообщения с токенами темы (surface, accent). Это frontend-логика — она не должна влиять на админ-визард, но если файл содержит ранние output/хуки без проверок is_admin(), может ломать.

Вероятный виновник в вашей теме → wh_enqueue_scripts (Tailwind v3.4.1 + Alpine.js + style.css). Если enqueue не ограничен !is_admin() (или wp_doing_ajax()), эти ассеты грузятся на wc-admin страницах → конфликты с React WC Admin, бесконечная загрузка, нерабочие кнопки. CSS-переменные (--wh-*) инжектятся в wp_head — проверьте, не затрагивает ли это админ (wp_head может срабатывать). Tailwind-классы/пост-обработка тоже могут ломать стили визарда.
Рекомендации по исправлению (по шагам)

Конфликт-тест:
Переключитесь на дефолтную тему WordPress (Twenty Twenty-Five / Twenty Twenty-One) или Storefront (рекомендуется WooCommerce).
Деактивируйте все плагины кроме WooCommerce (и Memberships, если установлен).
Попробуйте запустить визард заново. Если заработало — проблема в теме/плагинах. Включайте по одному.

Проверьте браузер и логи:
Откройте визард → F12 → Console/Network → ищите красные ошибки, failed requests (особенно REST /wc-admin/*).
WooCommerce → Status → System Status → проверьте ошибки, REST API, Page setup.

Технические фиксы:
Пермалинки → Настройки → Постоянные ссылки → Сохранить (flush rewrites). На IIS/специфических хостингах добавьте правила для /wp-json/.
Временно отключите security-плагины (Wordfence, firewall).
Skip/перезапуск визарда → WooCommerce → Help (в правом верхнем) → Setup Wizard. Или вручную перейдите на /wp-admin/admin.php?page=wc-admin (иногда пропускает).
Убедитесь, что в functions.php enqueue скриптов/стилей темы имеет условие:PHPif (!is_admin() && !wp_doing_ajax()) { /* enqueue Tailwind, Alpine, style.css */ }Удалите/закомментируйте Alpine/Tailwind в админке для теста.

Для Memberships:
Проверьте inc/memberships.php — нет ли ранних echo/output, конфликтующих хуков или admin-действий без проверок.
Используйте хуки из официальной документации (frontend/admin reference), чтобы избежать переопределений.

Дополнительно:
Очистите кэш браузера/сайта, отключите любые оптимизаторы (WP Rocket и т.п.) временно.
Если на Windows/IIS — проверьте web.config для URL rewrite (REST API routing).
После фикса визарда настройте магазин вручную (WooCommerce → Settings).


Если после дефолтной темы проблема остаётся — укажите ошибки из консоли, WooCommerce Status report или сервер (Apache/Nginx/IIS, PHP-версия) — это поможет точнее диагностировать. Часто именно тема с Alpine/Tailwind без изоляции админки вызывает такие симптомы. Удачи!
